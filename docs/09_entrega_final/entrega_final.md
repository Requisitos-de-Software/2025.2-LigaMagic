# Entrega Final - Grupo 2

## Introdução
A entrega final tem como objetivo consolidar todo o trabalho desenvolvido ao longo do projeto de Requisitos de Software, incluindo as etapas de elicitação, análise, especificação e validação dos requisitos.
Nesta fase, o foco está na verificação e validação (V&V) dos artefatos produzidos, garantindo que os requisitos definidos estejam corretos, completos e consistentes com as necessidades dos usuários do sistema analisado.


## Objetivo
Esta entrega visa:

- Verificar os artefatos produzidos nas etapas anteriores, assegurando a conformidade com os padrões definidos.

- Validar os requisitos junto aos usuários por meio de protótipos, gravações de feedback e comprovação informal.

- Avaliar a consistência, rastreabilidade e completude dos requisitos implementados e não implementados.

- Consolidar as evidências de verificação (checklists) e validação (interações com o usuário).

## Metodologia
Para a realização desta entrega, as seguintes etapas foram seguidas:

1. Revisão e verificação dos artefatos anteriores
    - Aplicação das listas de verificação (checklists) para os documentos produzidos: Rich Picture, Perfil do Usuário, Elicitação, Priorização, Cenários, Léxicos, Casos de Uso, Especificação Suplementar, Histórias de Usuário, Backlog, NFR Framework, Matriz de Rastreabilidade e Elos.
    - Cada item foi acompanhado de evidências (fotos, prints e hiperlinks) para comprovar a fonte e a revisão.

2. Validação com o usuário
    - Foram elaborados protótipos funcionais com base nos Requisitos Funcionais Não Implementados (RFNI).
    - Os protótipos foram apresentados aos usuários para coleta de feedback, com gravação do processo de validação e registro dos resultados.
